## 一、架构终版：五层 n8n 中枢调度模型

```
外部世界 (用户/定时器/Webhook/回调)
    ├─ 人的请求 → L4 Dify (:5001)
    └─ 系统事件 → L2 n8n  (:5678) 【唯一中枢调度器】
                       │
          ┌────────────┼────────────┬────────────┐
          ▼            ▼            ▼            ▼
       L4 Dify      L3 LangChain  L1 Claude    L5 OpenClaw
       :5001        :4010         :4020        :19001
       知识库RAG     Agent编排     代码/部署    通信/RPA
          │            │            │            │
          └────────────┴────────────┴────────────┘
                       │ 回调 n8n
                       ▼
                  Task Callback Hub
                       │
                       ▼
                 返回 + 通知 + 存档
```

| 层级 | 组件 | 端口 | 刚性红线 |
|:--:|------|:--:|------|
| L4 | Dify | :5001 | 不做复杂Agent，不定时调度，复杂任务上抛 n8n |
| L2 | n8n (中枢) | :5678 | 不做LLM推理，不做知识库，唯一有权编排跨组件任务 |
| L3 | LangChain | :4010 | 不面向用户，不配置可视化，不定时调度，不调度其他组件 |
| L5 | OpenClaw | :19001 | 不做AI生成，不编排工作流，不做代码变更 |
| L1 | Claude Code | :4020 | 不直接对接前端，不收终端用户提问，不做工作流调度 |

## 五、四条标准主链路

**链路A - 简易问答**：用户 → Dify → Claude → 返回 (复杂度<3，毫秒级)

**链路B - 复杂Agent**：用户 → Dify → n8n 中枢调度 → LangChain/Dify/Claude → 回调n8n → 返回

**链路C - 业务自动化**：n8n 定时触发 → Dify 生成 → LangChain 评测 → 入库 → OpenClaw 通知

**链路D - 研发变更**：n8n→Claude→安全审计→OpenClaw预检→部署→n8n复测→通知
