> 基于 `sz_mobile_data_card` 电销技能的全面扩展
> 版本：v3.0 | 日期：2026-05-16

---

## 一、现有产品解读

### 1.1 当前架构

```
号码池 → AI外呼引擎(OpenClaw Skill) → 话术节点流转 → 结果推送 → 企微机器人通知
                                    ↓
                               CSV存储(基本CRM)
```

### 1.2 现有能力清单

| 模块 | 能力 | 评价 |
|------|------|------|
| 话术引擎 | 16个意图识别 + 13个对话节点 + 3层挽留 | ✅ 完善，但意图判断较浅 |
| TTS语音 | Edge TTS (Xiaoxiao) | ⚠️ 仅云端TTS，无拟人停顿/呼吸 |
| ASR识别 | 本地语音识别 | ⚠️ 基础版，无语义理解 |
| 号码管理 | 4路深圳号码 + 黑名单 + 区域过滤 | ✅ 基础功能完整 |
| 外呼控制 | 定时/并发/重拨 | ✅ 基础功能完整 |
| 线索推送 | 企微机器人 webhook | ❌ 仅通知，无CRM |
| 桌面微信 | **无** | ❌ **核心缺失功能** |
| 通话分析 | **无** | ❌ **核心缺失功能** |
| 客户跟进 | **无** | ❌ **核心缺失功能** |

### 1.3 核心痛点与机会

| 痛点 | 影响 | 机会 |
|------|------|------|
| 只有企微推送，没有桌面微信加人 | 客户线索无法沉淀到个人微信 | 桌面微信自动化可打通私域 |
| 无通话内容分析 | 不知道客户真正关心什么 | NLP分析挖掘客户画像 |
| CRM只有CSV | 无法做客户分层、自动化跟进 | 构建完整的客户管理系统 |
| 无自动跟进 | 成交率依赖一次通话 | 微信自动跟进持续转化 |

### 2.2 模块一：AI 外呼引擎增强 ⬆️

#### 2.2.1 拟人化语音升级

| 特性 | 当前 | 升级后 |
|------|------|--------|
| TTS引擎 | Edge TTS | **Azure Neural TTS / 火山引擎** - 情感可控 |
| 语速控制 | 固定 | 动态调整（兴奋时稍快，确认时放缓） |
| 呼吸停顿 | 无 | NLP分析断句，插入自然呼吸/思考停顿 |
| 语气变化 | 单一 | 根据对话情绪自动切换（亲切/认真/兴奋/安抚） |
| 方言支持 | 普通话 | 支持粤语/四川话等（深圳场景常用） |
| 打断处理 | 超时静音 | **主动打断** - 客户说话时AI自动停止并倾听 |
| 笑声/语气词 | 无 | 自然插入"嗯嗯~"、"好的呢~"、"哈哈确实" |
| 背景音 | 无 | 轻微办公环境白噪音（可选） |

#### 2.2.2 对话引擎增强

**从"IF-THEN 话术树"升级为"LLM + 话术模板混合驱动"：**

```
传统模式（当前）：
  客户说"太贵了" → 命中 INT_MONEY → 播放固定应答
  ❌ 无法应对复杂问题
  ❌ 话术死板，容易被识别为AI

混合模式（升级后）：
  客户说"太贵了，别家才9块钱" →
  [1] 意图识别：INT_MONEY + 价格对比
  [2] LLM 实时生成：理解竞品信息 + 针对性应答
  [3] 话术模板兜底：如LLM失败则用预设话术
  ✅ 灵活应对各种场景
  ✅ 对话自然，拟人度高
```

**新增对话能力：**

| 能力 | 说明 |
|------|------|
| 多轮对话记忆 | 记住客户前面说过的话，避免重复追问 |
| 上下文关联 | "我刚才说了，这个套餐..." → 能接住 |
| 反客为主引导 | 客户沉默时主动抛选择题："您是担心费用问题还是使用问题？" |
| 异议深度解析 | "我不需要" → 分析是真的不需要还是托词 |
| 三步逼单法 | 价值确认 → 痛点放大 → 限时促单 |
| A/B话术测试 | 同一节点自动测试2版话术，统计转化率 |

        add_btn.Click()

        # 5. 填写验证信息 + 备注 + 标签

    def send_followup(self, contact_name: str, template: str):
        """根据模板发送跟进消息"""
        # 1. 搜索联系人
