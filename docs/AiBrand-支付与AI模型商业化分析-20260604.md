# AiBrand 支付模块与 AI 模型商业化分析

> 2026-06-04 · 支付板块如何支撑"对接各大模型厂商"的商业模式

---

## 一、现状：积分系统 vs 真实 AI 成本

### 当前实现

```
用户支付 RMB → 购买积分(credits) → 消费积分 → 调用 AI 模型
                  ↑                              ↑
              有充值系统                   没有计费系统
```

**有的**：
- 积分余额管理（MongoDB 存储，BigNumber 精度）
- 充值订单系统（16位订单号、支付状态机、过期处理）
- 积分流水记录（增/减/过期）
- 积分过期定时任务（每 10 分钟检查）

**没有的**：
- ❌ **AI 调用计费** — 用户调用一次 Agent 对话扣多少积分？没有定义
- ❌ **不同模型不同定价** — DeepSeek vs GPT-5 vs 图片生成，成本差 100 倍
- ❌ **成本核算** — 不知道每次 AI 调用花了多少钱
- ❌ **配额联动** — 订阅计划的"每月 N 次"和积分余额没有打通

### 这就是核心风险

你卖的是"AI 能力"，但 AI 的成本你不掌握。如果用户大量使用，你的成本是黑盒。

## 三、商业模式的三个致命问题

### 问题 1：积分定价没有锚点

当前 Free/Pro/Enterprise 三档订阅 + 积分充值，但**积分和 AI 成本之间没有映射关系**。

```
Free:    0 元/月, 每月 10 次 AI 操作
Pro:    ¥99/月, 每月 100 次
Enterprise: 按需

但"1 次 AI 操作"是什么？
- 一句话回复？
- 一篇长文创作？
- 一次带思维链的诊断？
- 一张图片生成？

不同操作的成本差 100 倍，不能统一定价。
```

### 问题 2：AI 成本不可控

如果用户大量使用最贵的模型，你的利润是负数。

```
假设 Pro 用户 ¥99/月，每天用 50 次 GPT-5 创作：
  你的收入：¥99/月
  你的成本：50次 × ¥0.05 × 30天 = ¥75/月
  毛利：¥24/月（24%）

同一个用户换 DeepSeek：
  成本：50次 × ¥0.003 × 30天 = ¥4.5/月
  毛利：¥94.5/月（95%）
```

**没有模型路由策略，你的利润率完全不可控。**

### 问题 3：模型厂商的定价会变

DeepSeek 现在便宜，明天可能涨价。GPT-5 现在贵，明天可能降价。你的积分定价一旦固定，成本侧的任何变化都会吃掉利润。

## 四、解决方案：三层计费架构

### 架构

```
用户支付 RMB
    ↓
按模型类型转换成"Token 额度"
    ↓
每次 AI 调用 → 按实际 Token 消耗扣费
    ↓
成本核算 → 自动路由到最优模型
```

### 第一层：Token 额度（不是积分）

**弃用"积分"概念，改用"AI Token 额度"。**

```
¥1 = 100,000 tokens (DeepSeek 基准)

用户充值 ¥100 → 获得 10,000,000 tokens 额度

使用时：
  调用 DeepSeek Chat → 消耗 1x tokens
  调用 GPT-5 → 消耗 15x tokens（模型价格系数）
  调用 FLUX 生图 → 消耗固定 tokens（图片定价系数）
```

**模型价格系数**（相对于 DeepSeek Chat 基准）：

| 模型 | 系数 | 说明 |
|------|:--:|------|
| DeepSeek Chat | 1x | 基准 |
| DeepSeek Reasoner | 4x | 含思维链 |
| GPT-4o | 10x | 高质量 |
| GPT-5 | 15x | 最新 |
| Claude Opus 4 | 15x | 深度推理 |
| Gemini 2.5 Pro | 5x | 多模态 |
| FLUX.1 Pro (1张) | 100x | 固定 |
| Kling 视频 (1秒) | 200x | 固定 |

### 第二层：智能模型路由

```
用户说"帮我写一篇小红书笔记"
    ↓
Agent 判断：这是短文案创作，不需要深度推理
    ↓
自动路由到 DeepSeek Chat（成本最低，质量够用）
    ↓
用户从来没有选过模型——Agent 自动选

用户说"帮我深度分析这篇竞品报告"
    ↓
Agent 判断：需要深度推理 + 长上下文
    ↓
自动路由到 Claude/GPT-5
    ↓
用户看到的是"Agent 在深度思考"，不关心底层用了什么模型
```

**路由策略**：
1. 简单创作（短文案、标题）→ DeepSeek Chat
2. 深度分析（报告、诊断）→ GPT-5/Claude
3. 多模态（图片理解）→ Gemini
4. 图片生成 → FLUX
5. 用户手动选择 → 支付对应系数

### 第三层：成本实时监控

```typescript
// 每次 AI 调用记录
interface AICallRecord {
  userId: string
  model: string
  inputTokens: number
  outputTokens: number
  costInTokens: number    // 扣除的 token 额度
  costInRMB: number       // 实际 API 花费
  margin: number          // 毛利
  timestamp: Date
}
```

**关键指标**：
- 每用户日均成本
- 每模型调用量/成本占比
- 整体利润率（收入/成本）
- 异常用量告警

## 五、对商业化的影响

### 🔴 如果不改

| 风险 | 影响 |
|------|------|
| 统一定价 | 用户全选最贵模型 → 亏损 |
| 无成本核算 | 不知道赚了还是亏了 |
| 模型不可替换 | 绑死 DeepSeek，它涨价你就涨 |
| 积分无锚点 | 用户不理解"1 积分 = 什么" |

### 🟢 改了之后

| 能力 | 商业价值 |
|------|---------|
| 模型差价赚钱 | DeepSeek 成本 1x，收用户 5-15x（用户不需要知道底层用什么） |
| 路由优化利润 | 自动选便宜模型做简单任务，利润率 90%+ |
| 高端用户付溢价 | 需要 GPT-5 的手动切换，付 15x 系数 |
| 成本可控 | 实时监控，异常自动限流 |
| 多模型冗余 | DeepSeek 挂了自动切 GPT，用户无感 |

## 总结

**支付模块不是"收钱"那么简单——它是 AI 模型商业化的定价引擎。**

当前积分系统是"传统 SaaS 的充值思维"，不适用于 AI 产品的成本结构。核心改造：**积分 → Token 额度 + 模型价格系数 + 智能路由 = 可控的利润率。**
