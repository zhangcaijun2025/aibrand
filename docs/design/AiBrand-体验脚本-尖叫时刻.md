# AiBrand 体验脚本 — "尖叫时刻"

> 不是功能说明，是用户内心的真实感受。

---

## 脚本1：第一次相遇

```
用户打开 AiBrand。

黑色屏幕中央，只有一行字，逐字浮现：

"正在连接你的 AI 运营中枢..."

[1.5秒后]

"晚上好。

全部 7 个组件在线。
你的 3 个账号今天新增 127 条互动。
有 2 条竞品动态值得关注。

你想先看哪个？"

用户内心：它怎么知道我有3个账号？它主动告诉我该做什么，不是等我去找。
```

---

## 脚本2：一句话的力量

```
用户输入："看看最近竞品在做什么"

Agent 逐行呈现思考过程：
  → 搜索知识库...[完成 ✓]
  → 调用 Dify AI 分析...[完成 ✓]
  → 触发竞品抓取工作流...[已触发]

[15秒后]

Agent：
"过去7天，你的3个主要竞品动态：

A品牌：下调了Pro版价格 15% → 建议关注你的定价策略
B品牌：发布了AI客服新功能 → 建议评估差异化应对
C品牌：小红书投放量翻倍 → 建议分析其ROI

我已生成详细对比报告。要现在看吗？"

用户内心：我就说了一句话。它自动判断要查知识库、要分析、要抓数据。我什么都没做，报告有了。
```

---

## 脚本3：它在我发现之前就修好了

```
凌晨3点。用户在睡觉。

n8n 服务响应延迟异常。
→ 自愈引擎检测到
→ 匹配剧本 rb-002
→ 自动重启 n8n
→ 验证恢复（延迟 3.2s → 0.04s）
→ 记录进化日志

早上9点。用户打开 AiBrand：

"早上好。昨晚发生了一件事想告诉你。

凌晨3:07，n8n 服务出现延迟异常。
我自动重启了它，3秒后恢复。

这是本月的第2次类似故障。
我已分析了根因并生成了预防方案，要审批吗？"

用户内心：它自己发现问题、自己修好、还给了我预防建议。我什么都没做，问题已经解决了。
```

---

## 脚本4：它记得我

```
用户第七次打开 AiBrand：

"欢迎回来。

你上周的竞品分析关注了A、B、C三个品牌。
A品牌今天有新动态——他们刚刚发布了Q2财报。
B品牌的定价页面发生了变化。

你之前提到的618营销计划，距离预热期还有3天。
我已生成了预热内容初稿。

继续上周的工作，还是先看竞品动态？"

用户内心：它记得我上周在做什么。不是"重新开始"，是"继续推进"。
```

---

## 脚本5：它帮我发现了新机会

```
用户在浏览竞品报告。Agent 主动插入：

"打断一下。我在分析竞品数据时发现了一个有意思的事：

你的竞品在小红书的互动率平均是3.5%，你的是2.1%。
但我进一步分析后发现——你在B站的互动率是4.8%，远超竞品的2.1%。

建议：你的优势在B站，要不要加大B站投放？我可以帮你调整内容策略。"

用户内心：它不只是回我问题，它在帮我发现我不知道的事。
```

---

## 脚本6：进化是可感知的

```
用户第N次打开 AiBrand：

"晚上好。今天有件事想告诉你。

过去两周，我发现你经常需要做竞品分析。
我自动生成了一个快捷技能「竞品周报」。

从今天开始，每周一早上8点，我会自动生成竞品周报。
不需要你每次手动触发。

这是我过去7天学到的3件事：
1. 你的竞品分析偏好关注「定价」和「新功能发布」
2. 你更喜欢表格形式而非段落
3. 你通常在上午处理策略类工作

要我在这些方面优化吗？"

用户内心：它在进化。不是固定功能，是在越变越好。
```

---

## 体验铁律

| # | 原则 | 反面 |
|:--:|------|------|
| 1 | Agent 主动说话，不等用户问 | ❌ 空白界面等待用户输入 |
| 2 | 每句话都有价值，不废话 | ❌ "你好，我是AI助手" |
| 3 | 步骤可见但不啰嗦 | ❌ 后台日志式的技术细节 |
| 4 | 记得上次的事，不是重新开始 | ❌ 每次都是全新对话 |
| 5 | 发现用户不知道的事 | ❌ 只回答不洞察 |
| 6 | 进化是可感知的 | ❌ 改了但用户不知道 |
| 7 | 修复是悄悄的，但结果要告知 | ❌ 修了不说，或修了报一堆技术日志 |
