## 前言：重新定义「运营工具」

市面上的运营工具都是**工具**——你打开网页，看到表单，填内容，点按钮，等结果。用完就关，下次打开再重复。它们是**死的**。

AiBrand 是**活的**。

你不是打开一个工具，你是**唤醒一个等待你的 AI 伙伴**。它会呼吸。它会记得你是谁。它会在你离开时继续工作，等你回来时汇报发生了什么。它越了解你，就越像你的专属运营合伙人。

这不是一个比喻。这是代码里已经实现了的状态机、感知模型、演化引擎和苏醒动画。

## 一、产品灵魂：你不是在使用工具，你在唤醒伙伴

### 1.1 苏醒仪式（The Awakening Ritual）

你打开 AiBrand。屏幕是一片深邃的暗色。一个光点开始呼吸。

这不是加载页面——**这是苏醒**。

```
IDLE → WAKING → READY（或 SLOW / DEGRADED）
```

**600ms**：呼吸光点浮现，以 `framer-motion` 驱动的弹性缩放动画呼吸。它像心跳一样规律。你的 AI 伙伴正在启动。

**1200ms**：光点展开成一条水平线。不是冷冰冰的进度条，而是一只睁开的眼睛。

**2000ms**：问候文字逐字显现。「晚上好，XIAOMI。欢迎回来。」

**3000ms**：卡片浮现。一个简报系统告诉你：系统运行正常，你有 3 个项目在推进，追踪到 2 个竞品的动态，距离上次访问已经过了 36 小时。

**4000ms**：所有动画完成。环境光点回到右下角，保持柔和的脉冲呼吸。你的 AI 伙伴就绪了。

但这只是理想路径。系统有三种异常处理：

- **SLOW（>5s）**：数据加载较慢时，页面保持优美的脉冲动画，显示「调取记忆中...」。不会报错，不跳失败页，只是优雅地告诉你它在努力。
- **DEGRADED（>8s）**：如果服务真的不可用，页面会降级。不是白屏，不是 500 错误——而是一个精心设计的降级态，告诉你发生了什么，提供重试按钮。
- **Space 跳过**：任何时候按空格键，跳过所有动画，直达就绪状态。尊重效率型用户的习惯。

这不是设计师凭空想象的，是 `AgentGreetingCore.tsx` 里 575 行代码实现的完整苏醒状态机。

### 1.2 三层感知模型

苏醒只是开始。真正让 AiBrand「活的」的，是后端的三层感知架构——**这不是 AI 的套壳，而是 AI 的记忆系统**。

| 感知层 | 存储 | 内容 | 生命周期 |
|--------|------|------|----------|
| **L1：系统感知** | SystemEvent (MongoDB) | 系统状态、故障恢复、里程碑事件 | 实时 |
| **L2：用户上下文** | UserContext (MongoDB) | 追踪的竞品、活跃项目、近期会话、偏好的平台/风格/格式 | 持续更新 |
| **L3：用户画像** | UserProfile (MongoDB) | 行业、角色、总创作量、里程碑、擅长领域、规避话题、新手引导状态 | 长期演化 |

当你使用 AiBrand，你的每一个行为都会被优雅地记录和分析：

- 你选了什么行业？（餐饮 / 时尚 / 科技 / 旅行 / 生活方式）
- 你偏好在哪些平台发布？（抖音 / 小红书 / B站 / YouTube）
- 你喜欢什么内容格式？（短视频 / 图文笔记 / 长文章）
- 你习惯什么时候工作？（上午 9 点 / 深夜 11 点）
- 你拒绝了哪些建议？（这个话题我不感兴趣，别再推荐了）

24 个 API 端点、4 个 MongoDB Schema、3,500+ 行后端代码，构成了 Agent 的大脑。这不是从 ChatGPT 套壳来的，是自研的。

## 二、核心交互：Agent 对话系统

### 2.1 SSE 流式对话——每一步都看得见

AiBrand 的 Agent 对话不是「发消息→等回复」的黑盒。它是**分步骤、可观察的协作过程**。

当你对 Agent 说「帮我分析竞品最近一周的内容策略」，你看到的不是一条 loading 动画然后弹出结果。你看到的是：

```
🔍 搜索知识库 —— ✓ 完成
📊 抓取竞品内容 —— ⏳ 进行中
🧠 分析策略变化 —— 等待中
📝 生成对比报告 —— 等待中
```

每一步都是一个带状态指示器的可展开卡片。下面是实时刷新的文字流：

```
正在分析「花西子」近7天的小红书内容...
发现 3 个内容趋势：
1. 国风联名频率上升 40%
2. 视频笔记占比从 30%→65%
3. 互动率峰值出现在周五 20:00→22:00
```

这种体验的技术实现非常复杂但设计上非常克制：

- **每对话任务独立实例**：`TaskInstance` 类管理每个对话的完整生命周期（流式缓冲、步骤索引、当前消息 ID），多个对话并行执行互不干扰
- **Dify 事件翻译层**：后端 NestJS 服务接收 Dify 的事件流（`agent_thought`、`message`、`message_end`），翻译成前端友好的事件类型（`step_start`、`step_done`、`message`、`done`）
- **工具名中文映射**：`knowledge_retrieval` → 「搜索知识库」，`web_scraping` → 「网络抓取」，让非技术用户一目了然
- **Redis Pub/Sub 中断通道**：你可以随时中断 Agent 的任务，信号通过 Redis 发布/订阅机制传递到后端，立即停止 LLM 调用

### 2.2 Action Cards——Agent 不只是聊天

对话中的 Agent 不仅回复文字，还能**主动发起操作**。

它会在消息末尾推出一张 Action Card：

| 卡片类型 | 示例 | 行为 |
|----------|------|------|
| **一键发布** | 「好的，内容已生成。要发布到抖音吗？」 [确认发布] | 触发多平台发布流程 |
| **保存草稿** | 「我把这版保存到草稿箱？」 [保存] | 写入草稿箱 |
| **修改重试** | 「标题有敏感词，已为你修改。满意吗？」 [重新生成] [确认] | 支持分支操作 |
| **充值提示** | 「你的 Token 余额不足，需要充值吗？」 [去充值] | 跳转充值页 |

这是 Strategy 模式实现的 `ActionHandler` 注册表——当 Agent 返回特定 `action` 字段，前端自动匹配处理函数，无需用户理解背后的复杂流程。

### 2.3 情绪化思考动画

Agent 「思考中」的动画不是千篇一律的转圈：

- **思考指示器**：三段彩色光点（蓝→紫→粉），以 `thinkingGlow` 关键帧循环——色相每 1.8 秒旋转 180°，模拟「脑电波」效果
- **打字指示器**：三段不同延迟的弹跳点（0s / 0.15s / 0.3s），自然模拟人类敲键盘节奏
- **工具状态动画**：`Loader2` SVG 旋转 + `CheckCircle2` 完成标记，每一步都有清晰的视觉反馈

## 三、Evolution Engine：越用越懂你的平台

### 3.1 行为学习系统

AiBrand 不只是一个工具集合，它是一个**持续学习你偏好的演化系统**。

**四因子加权排序**——每次你使用平台，系统都在分析：

| 因子 | 权重 | 含义 |
|------|:--:|------|
| 使用频率 | 40% | 你每天用这个功能，说明它对你重要 |
| 使用新近度 | 30% | 你最近开始用的功能，权重快速上升 |
| 上下文关联 | 20% | 你在这个场景下习惯用这个功能 |
| 任务完成率 | 10% | 你用这个功能完成了多少任务 |

趋势检测（上升/稳定/下降）自动标记每个模块的热度变化。一个新功能从 0 到高频使用，系统会在约 3-5 次行为后感知到变化。

### 3.2 自适应布局

当你的使用模式发生显著变化（Kendall Tau 距离 > 30%），系统会自动建议重新排列你的工作台布局。不是强制重排——是一条优雅的提示：「我们发现你的工作习惯发生了变化，要调整布局吗？」

**Kendall Tau 算法**：一种非参数统计方法，用于衡量两个排列的一致性。在这里它被用来检测你的功能使用排序和当前布局排序之间的偏差程度。

### 3.3 自动偏好学习

系统会自动从你的行为中学习：

- **拒绝检测**：你连续 3 次拒绝了「美食」话题的建议 → 自动加入 `avoidTopics` 列表，下轮不再推荐
- **平台偏好**：你 80% 的发布去抖音和小红书 → 默认选项优先展示这两个平台
- **工作时间**：你的高效创作集中在晚上 9-11 点 → 定时发布建议自动匹配这个窗口
- **内容风格**：你的标题偏短、直接、有冲击力 → AI 生成时自动适配这个风格

### 3.4 每周演化报告

每周一早晨，当你唤醒 Agent 时，它会告诉你：

> 「上周你创作了 23 篇内容，比前一周多 40%。小红书的数据最好——平均互动率 5.2%，超过你同品类 90% 的创作者。我注意到你现在更多用视频而不是图文了，需要我帮你调整内容策略吗？」

这是 `EvolutionService.generateEvolutionReport()` 方法每周自动运行的结果。它不是模板化的话术——每一句话都对应数据库里的实际行为数据。

## 四、Agent 精灵与组件市场

### 4.1 8 个预置 Agent，各有性格

AiBrand 预置了 8 个专业 Agent，各有分工：

| Agent | 角色 | 擅长 |
|-------|------|------|
| **内容策略师** | 选题策划 | 热点追踪、竞品分析、选题打分 |
| **文案大师** | 内容创作 | 标题优化、文案生成、多平台适配 |
| **视觉顾问** | 图片/视频 | 封面建议、排版检查、色彩推荐 |
| **数据分析师** | 数据洞察 | 趋势分析、异常检测、对比报告 |
| **发布管家** | 发布调度 | 最佳时间、多平台编排、频率控制 |
| **合规官** | 风险检测 | 敏感词过滤、平台规则检查、版权提醒 |
| **互动教练** | 客户沟通 | 评论回复建议、私信话术、危机应对 |
| **增长顾问** | 增长策略 | 涨粉策略、爆款分析、矩阵规划 |

每个 Agent 都有自己的 prompt 模板、可配置参数和触发场景。你可以一键开启/关闭、拖拽排序、或在 Agent 精灵页重新配置。

### 4.2 组件市场——Agent 的能力商店

Agent 的能力是**可插拔的组件**。

```
组件市场
├── 竞品分析组件 —— Agent 可以访问你的竞品数据
├── 定时发布组件 —— Agent 可以在指定时间自动发布
├── 舆情监控组件 —— Agent 可以实时追踪品牌关键词
├── AI 画图组件 —— Agent 可以调用 DALL·E 生成配图
└── 微信客服组件 —— Agent 可以接入企业微信回复客户
```

开启一个组件，相应 Agent 就多一项能力。关闭它，Agent 就回到默认状态。这就像给手机装 App——Agent 的能力边界由你定义。

`AgentDefinition` + `ComponentDefinition` + `UserInstalledComponent` 三个 MongoDB Schema 构成了这个系统的数据模型。Agent 有 `quickConfig`（一键配置）、`reset`（恢复默认）、`reorder`（拖拽排序）和 `match`（智能匹配推荐）四个操作。

### 4.3 内部 AI 控制台

为高级用户准备的开发者面板（路径：`/ai-console`）：

- **演化指数环**：SVG 环形进度图，实时显示你的 Agent 学习进度
- **粒子背景**：30 个漂浮青色粒子在 Canvas 上以 `requestAnimationFrame` 循环流动——不只是好看，每个粒子的速度和透明度代表不同的系统指标
- **命令面板**（⌘K）：开发者快捷键，直接与底层 Agent 交互
- **服务脉搏**：`animate-ping` 驱动的脉动指示灯，显示各 AI 服务的健康状态
- **演化指标**：Runbook 数量 + Skill 数量，衡量 Agent 的学习深度

## 五、微交互与情感设计

### 5.1 全平台微交互系统

AiBrand 的设计系统不只是颜色和字体——每一个交互细节都被精心设计。

**所有按钮**：`transition-all duration-200`。悬浮时阴影上升（`shadow-md`）。这不是普通的 hover 效果，而是给每个可点击元素赋予「呼吸感」——它们不是死的像素块，而是会对你鼠标的靠近做出反应的界面元素。

**卡片悬浮**（`.card-hover`）：`transition-all duration-300`。悬浮时阴影加深 + 向上平移 1px（`shadow-xl -translate-y-1`）。这个 1px 的位移极其微妙，但它制造了卡片「浮出水面」的错觉——人眼对这个微小的深度变化非常敏感。

**MorphingIcon**：一个不断变化的动态图标。四个 SVG 图案（✨火花 / 🧠大脑 / 🚀火箭 / 🎯准星）每 1.5 秒切换一次。不是硬切——`framer-motion` 的 `pathLength` 动画让每条 SVG 路径像被画笔描过一样逐渐显现。同时一个呼吸式的不透明度循环（2 秒周期）让图标「活」在界面上。

**AiAssistantWidget 脉冲环**：悬浮按钮的 `::after` 伪元素以 `pulse-ring` 动画持续呼吸——从 1x 缩放到 1.3x，同时透明度从 0.4 到 0。这不是装饰性的，它在告诉用户：「我在这里，我活着，随时可以帮你」。

**思考状态**：不是千篇一律的转圈 GIF。三个品牌色光点（蓝/紫/粉）以错开的节拍跳跃，梯度文字以色相旋转 180° 的动画循环。这些动画让人感觉 Agent 在「思考」，而不是加载。这是体验设计中最关键的区别。

**发布对话框**：不是突兀的弹出框，而是平滑的 300ms 面板过渡——调用 `cubic-bezier(0.16, 1, 0.3, 1)`，一个精心调校的缓出曲线，让面板「滑出」而不是「弹出」。

### 5.2 引导与发现

**4 步新手引导**（`Onboarding` 组件）：不是传统的遮罩层 + 箭头。而是一个诊断式对话——用户选择角色→评估痛点→标注成熟度→获得 AI 生成的升级路线图。

路线图包含三个时间维度：本周该做什么、本月该做什么、三个月内该做什么。每个步骤都是 AI 根据你的回答生成的，真实可用。

**driver.js 插件引导**：当用户首次触发浏览器扩展发布功能时，一个高亮圈指向插件按钮，说「点这里一键填充内容到平台」。1500ms 延迟出现，不打扰用户的当前操作流。

## 六、设计系统

### 6.1 OKLCH 色彩空间

AiBrand 的设计 Token 全部使用 **OKLCH 色彩空间**，而非传统的 Hex/RGB/HSL。

为什么？因为 OKLCH 是**感知均匀**的——两个颜色在 OKLCH 中的数值差异，对应人眼感知到的差异是线性的。传统 HSL 的「50% 明度」在不同色相下看起来明度完全不同。OKLCH 消除了这个问题。

```css
--brand-purple: oklch(55.8% 0.288 302.321);
--brand-cyan:   oklch(54.6% 0.245 262.881);
--brand-gradient: linear-gradient(135deg, var(--brand-purple) 0%, var(--brand-cyan) 100%);
```

### 6.2 品牌渐变系统

不只是两种颜色的渐变——而是一整套分层渐变工具：

| Token | 用途 | 效果 |
|-------|------|------|
| `--brand-gradient` | 主要按钮、CTA | 135° 紫→青 |
| `--brand-gradient-horizontal` | 横向装饰条 | 0° 紫→青 |
| `--brand-gradient-subtle` | 背景光晕 | 低透明度，大面积氛围 |
| `--brand-gradient-glow` | 激活态底色 | 叠加在选中项上，微妙的品牌光泽 |
| `--brand-shadow-sm/md/lg` | 阴影层级 | 品牌色调阴影，而非灰色 |

### 6.3 明暗双主题

两个主题的变量被完整定义，不只是反转颜色——暗色模式有专门的调色板：

**亮色模式**：纯白底色，紫色主色，青色点缀  
**暗色模式**：oklch(0.145) 深邃底色，明亮文字，适当调整的强调色

暗色模式不是简单反转。侧边栏、成功色、警告色、图表色在两种模式下保持一致。

### 6.4 品牌专属字体

全平台使用 **Suisseintl** 字体族，一种现代几何无衬线体，与品牌的技术感和未来感契合。
