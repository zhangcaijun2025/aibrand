# AiBrand — AI 全域运营平台 产品介绍

> 版本 1.0 | 2026-06-04 | 基于 AiBrand MVP 技术架构编写

---

## 一、产品概述

**AiBrand**（爱品牌）是一站式 AI 全域运营平台，专为超级个体、一人公司和中小微企业打造。平台以"AI 工作流驱动"为核心理念，将内容创作、多平台发布、数据追踪、客户互动全链路智能化，帮助用户用一个人的团队做出一个公司的运营效果。

从传统 SaaS 工具的"表单+按钮"模式，升级为 **AI 原生的智能运营引擎**——用户只需输入意图，AI 自动完成从选题研究到内容发布、再到效果追踪的完整闭环。

### 一句话定位

> **AI 全域运营引擎：选题→创作→优化→发布→AI 搜索推荐→数据回收，全链路 AI 驱动。**

## 二、核心功能模块

### 1. 智能内容工厂（核心模块，Phase 1 交付）

**6 步 AI 工作流闭环**，用户输入一句话意图，系统自动完成：

| 步骤 | 功能 | AI 驱动方式 |
|------|------|-------------|
| ① 意图分析 | LLM 解析行业/目标/平台/调性 | GPT / Claude 意图分类 |
| ② 策略研究 | 趋势洞察 + 竞品分析 + 历史数据回顾 | Dify RAG + n8n 抓取 |
| ③ 选题确认 | 生成 3 个选题方向，用户一键确认 | Dify Agent 创意生成 |
| ④ 内容生成 | 并行生成多平台适配版本 | Claude 创意生成 + 平台模板 |
| ⑤ 质量检测 | 合规检测 + 吸引力评分 + 原创度检验 | 规则引擎(90%) + LLM 复核(10%) |
| ⑥ 发布策略 | 最优发布时间 + 排序推荐 + 48h 数据回收 | n8n 自动化编排 |

**单条内容成本：¥0.30（优化后）**，含 LLM Token + 存储 + API 调用。

### 2. 多平台一键发布

支持 **14 个主流平台**，一键分发：

| 国内平台 | 国际平台 |
|----------|----------|
| 抖音 | TikTok |
| 小红书 | YouTube |
| 快手 | Instagram |
| B 站 (Bilibili) | Facebook |
| 微信视频号 | Twitter/X |
| 微信公众号 | LinkedIn |
| 百家号 | Pinterest |
| | Threads |

- **平台适配引擎**：自动调整标题长度、话题标签、图片尺寸、内容风格适配各平台规则
- **浏览器扩展注入**：通过 MultiPost 扩展在目标平台页面自动填充内容
- **OAuth 授权管理**：统一管理多平台账号授权状态，支持连接、续期、解绑

### 3. 品牌推广（二维码营销）

- 创建推广计划，AI 批量生成视频/图文素材
- 生成推广二维码，顾客扫码即可一键将内容发布到自己的社交账号
- 实时追踪浏览量、点赞、评论、分享数据
- **裂变式传播**：C 端用户成为品牌的分布式传播节点

### 4. AI 全媒诊疗

四大诊断维度，AI 驱动的内容质量保证体系：

| 诊断类型 | 功能 |
|----------|------|
| **内容诊断** | 单篇内容综合评分（文案质量/吸引力/合规性/平台适配度） |
| **优化处方** | 基于诊断结果生成具体优化方案，逐条可执行 |
| **智能复诊** | 已发布内容追踪 + 效果复查 + 持续改进建议 |
| **账号体检** | 五维健康评分（内容力/互动率/粉丝增长/爆款率/合规度） |

后端引擎：NoteRx AI 诊断服务（localhost:8000） + Dify Agent 多视角评审 + n8n 工作流自动同步 → Dify 知识库沉淀。

### 5. 客户互动管理

- **企业微信接入**：扫码一键绑定，客户消息直达平台
- **客户 360 视图**：互动历史、偏好标签、活跃时段
- **智能回复建议**：AI 理解客户意图，推荐回复话术
- **客户旅程追踪**：从初次触达到长期复购的全链路追踪

### 6. 数据洞察与 AI 搜索优化

- **跨平台数据聚合**：统一查看 14 个平台的内容表现
- **AI 搜索可见性追踪**（GEO 引擎）：追踪内容在 Perplexity / 秘塔 / Kimi / Google AI Overviews 中的排名
- **Schema 结构化输出**：自动为内容生成 AI 搜索引擎友好的结构化数据
- **48 小时效果报告**：发布后自动追踪 → 数据回收 → 对比分析 → 优化建议

### 7. Agent 智能助手

- **右下角悬浮 AI 助理**：24/7 在线，支持 FAQ 知识库即时问答
- **苏醒仪式**：类 JARVIS 风格的问候动画，三层感知（系统状态 / 用户上下文 / 自适应行为）
- **千人千面**：Evolution Engine 学习用户偏好（格式/话题/平台），自动调整布局和推荐
- **Agent 组件市场**：8 个预置专业 Agent + 4 个内置组件，支持自定义扩展

### 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| **前端框架** | Next.js 14 (App Router) | SSR/SSG、SEO 友好、文件路由 |
| **UI 组件** | Radix UI + shadcn + Ant Design 5 | 无样式冲突的高质量组件库组合 |
| **状态管理** | Zustand + React Hook Form | 轻量、类型安全、无 Boilerplate |
| **样式方案** | Tailwind CSS 4 + CSS Variables | Utility-first + 品牌设计 Token |
| **后端框架** | NestJS 11 (Nx Monorepo) | 模块化架构、依赖注入、企业级 |
| **数据库** | MongoDB (Mongoose) | 灵活 Schema、适合内容型应用 |
| **缓存** | Redis (ioredis) | 会话管理、热数据缓存、队列 |
| **文件存储** | RustFS (Rust 编写) | 高性能、S3 兼容 API |
| **API 网关** | Nginx | 反向代理、SSL 终结、静态资源 |
| **容器化** | Docker Compose | 17 容器统一编排、一键部署 |
| **AI 框架** | Dify Community Edition | RAG + Agent + 知识库 + 可视化编排 |
| **工作流引擎** | n8n (自托管) | 可视化工作流、300+ 集成节点 |
| **LLM 路由** | GPT-5-mini / Claude 4.6 / DeepSeek | 按步骤复杂度分级调用，优化成本 |
| **实时通信** | SSE (Server-Sent Events) | AI 工作流状态推送、Agent 对话流式输出 |

### Agent 感知层（24 API，8 commits，~3500 行）

AiBrand 区别于传统 SaaS 的核心差异化层：

| 阶段 | 功能 | 关键实现 |
|------|------|----------|
| **Phase A** | Agent 问候 | 4 MongoDB Schema (SystemEvent/UserContext/UserProfile/UserBehavior)，时间/姓名/系统状态/简报卡片聚合 |
| **Phase B** | Agent 对话 SSE | Dify 流式代理 + 事件翻译，工具名中文映射 |
| **Phase C** | 千人千面 | EvolutionService，四因子加权排序，Kendall Tau 布局调整检测 |
| **Phase D** | Agent 精灵 + 组件市场 | 8 个预置 Agent，Agent CRUD + quickConfig + reorder |

## 四、AI 平台引擎

### Dify 知识库体系（9 容器，3 知识库）

| 知识库 | 用途 | 数据源 |
|--------|------|--------|
| FAQ 知识库 | AiAssistant 即时问答 | 产品文档 + 常见问题 |
| 行业最佳实践 | 内容策略研究 | 行业报告 + 历史爆款 |
| 平台规则库 | 合规检测 + 平台适配 | 14 平台官方规则文档 |

### n8n 工作流（4 个自动化流程）

| 工作流 | 功能 | 触发方式 |
|--------|------|----------|
| 竞品分析 | 定时抓取竞品内容，分析策略变化 | 定时 (每日) |
| 热搜话题 | 抓取各平台热搜/趋势话题 | 定时 (每小时) |
| 发布追踪 | 内容发布后追踪表现数据 | Webhook (发布后触发) |
| 账号健康 | 定期检测账号健康指标 | 定时 (每周) |

### 中央调度层（NestJS）

- **统一 API 入口**：所有 AI 服务通过 NestJS API 统一调用
- **认证与授权**：JWT 认证 + 配额 Guard
- **工作流引擎**：编排 Dify + n8n + NoteRx 的调用链
- **配额与计费**：Token 配额替代条数配额，公平透明
- **SSE 代理**：Dify 流式输出通过 NestJS 代理转发

## 七、技术亮点

1. **Nx Monorepo 工程化**：前端 Next.js + 后端 NestJS 统一管理，共享类型定义
2. **17 容器 Docker 编排**：一键 `docker-compose up -d` 部署全栈服务
3. **Agent 感知层**：行业首创的 Agent 三层感知模型（系统状态→用户上下文→自适应行为）
4. **双层合规架构**：90% 规则引擎快速过滤 + 10% LLM 复核疑难 case，兼顾速度与准确率
5. **成本优化策略**：DeepSeek 替代 GPT-mini 做预处理 → 单条成本 $0.13→$0.04，毛利率 52%→85%
6. **SSE 流式体验**：AI 工作流逐步骤可见，从"黑盒等待"升级为"协作体验"
7. **数据闭环**：发布→追踪→分析→优化，48 小时完整数据回流
8. **Token 配额制**：替代传统条数配额，按实际 AI 消耗计费，公平透明
