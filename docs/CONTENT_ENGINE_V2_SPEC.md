## 一、核心设计哲学

**一句话总结**：前端展示选择题卡片（小 A），背后驱动智能跳转的决策树（小 D），最终产出结构化 Brief（小 D Schema），整个流程在 AI 对话中沉浸式完成（小 A）。

**核心转变**：从「要求用户精准写提示词」变为「AI 主动挖掘意图 + 补齐素材 + 守护质量」。用户只需要做选择题和简单调整。

**三层协作关系**：

```
用户模糊输入
    ↓
[小A] 对话式引导卡片（前端交互层）
    ↓
[小D] 动态采访路由引擎（后端逻辑层）
    ↓
[小D+小C] 结构化 ContentBrief Schema（数据契约层）
    ↓
分发至文案/图片/视频/剪辑模块
```

## 四、分模块详细规格

### 4.1 意图理解层【P0，第 1-2 周】

**核心职责**：把用户模糊输入转化为结构化 Brief，是所有下游模块的数据源头。

#### 4.1.1 智能路由入口

```
用户输入 → LLM 快速分类（< 500ms，轻量模型）
  ├── 意图清晰（含产品+平台+风格 3/5 要素，clarityScore ≥ 0.7）
  │     → 直接生成 Brief 确认卡片，用户一键确认或微调
  │
  ├── 意图模糊（含 1-2 个要素，0.3 ≤ clarityScore < 0.7）
  │     → 补齐式提问，只问缺失的 1-3 轮选择题
  │
  ├── 意图极模糊（clarityScore < 0.3，"帮我想想发什么"）
  │     → AI 主动推荐方向（热点+竞品+历史数据）→ 用户从中选
  │
  └── 用户明确说"我自己填"
        → 展开五维表单（保留原有功能）
```

#### 4.1.2 动态采访决策树

**不是固定 5 轮，而是「最多 5 轮，智能跳过」**：

```
[检查品牌知识库 + 账号绑定 + 历史 Brief]
          ↓
  已获取的信息自动跳过
          ↓
[卡片 1] 行业品类（已从品牌库获取 → 跳过）
[卡片 2] 目标人群（已从品牌库获取 → 跳过）
[卡片 3] 内容目的（必问，或用户已在初始输入中说明 → 跳过）
[卡片 4] 内容风格（品牌库有默认 → 展示确认，一键采纳）
[卡片 5] 发布平台（已绑定账号 → 自动填入，可增删）
          ↓
    生成结构化 Brief → 展示确认卡片
```

#### 字段来源矩阵（智能跳过规则）

| 字段 | 自动获取来源 | 跳过条件 | 需用户确认 |
|------|------------|---------|------------|
| industry | 品牌知识库 > 历史 Brief > URL 提取 | 已有且置信度 > 0.8 | 低置信度时给选项确认 |
| targetAudience | 品牌知识库 | 已有值 | 每次创作可调整 |
| contentType | **无法自动获取** | 从不跳过 | 每次必问（或用户主动指定） |
| style.tone | 品牌知识库 > 历史偏好 | 品牌库已有 | 提供默认值，一键采纳 |
| style.visualStyle | 品牌知识库 | 品牌库已有 | 提供缩略图确认 |
| platforms | 已绑定账号 | 已绑定的自动填入 | 可增删 |
| product.* | 品牌知识库 > URL 提取 > 历史 Brief | 产品信息完整 | 可修改 |
| brand.* | 品牌知识库（首次入库后永久复用） | 全部已有 | 首次录入需填写 |

#### 4.1.3 行业粒度自适应

```
大类行业（1 轮）        细分行业（需追问）
─────────────────────────────────────────
餐饮                →  火锅 / 茶饮 / 快餐 / 烘焙 / ...
服饰                →  女装 / 男装 / 运动 / 童装 / ...
SaaS                →  CRM / 财税 / HR / 协同办公 / 营销工具 / ...
美妆                →  护肤 / 彩妆 / 香水 / 个护 / ...
教育                →  K12 / 职业教育 / 语言培训 / 兴趣教育 / ...
```

触发追问条件：`industryDepth === 'coarse' && categoryNeedsRefinement(industry.primary)`

#### 4.1.4 样本反推选型（原方案保留）

用户无法描述风格时：
1. 系统展示同行业 20 条成品案例
2. 用户勾选喜欢的内容（可多选）
3. AI 提取风格向量 → 绑定到 Brief 的 style 字段
4. 后续生成自动应用该风格

#### 4.1.5 前端组件：InterviewCard

嵌入对话流的卡片组件，支持三种模式：

```typescript
interface InterviewCardProps {
  mode: 'single_select' | 'multi_select' | 'text_input';
  question: string;
  subtitle?: string;          // "已从你的品牌信息自动填入，可修改"
  options?: Array<{
    label: string;
    value: string;
    thumbnail?: string;       // 可选缩略图
    preselected?: boolean;    // 是否预选（智能跳过用）
  }>;
  placeholder?: string;       // text_input 模式
  examples?: string[];        // 输入示例
  currentStep: number;        // "第 2/5 步"
  allowSkip: boolean;
  onAnswer: (value: any) => void;
  onSkip: () => void;
}
```

### 4.2 文案生成模块【P1，第 3-5 周】

**核心职责**：基于结构化 Brief 生成高质量、去 AI 味、平台适配的多版本差异化文案。

#### 4.2.1 三段式生成流水线

```
[结构化 Brief]（来自意图理解层）
        ↓
[阶段 1] 策略层（n8n 自动预取，生成前完成）
  ├── 竞品同赛道近期爆款数据分析
  ├── 所选平台实时热点趋势
  └── 用户品牌档案查询（品牌人设、禁用词、固定话术）
        ↓
[阶段 2] 创意层（Dify + DeepSeek）
  ├── 同一产品自动输出 5 种差异化创作角度
  ├── 内置文案结构库：
  │   ├── AIDA（注意→兴趣→欲望→行动）
  │   ├── PAS（问题→焦虑→解决方案）
  │   ├── 故事弧线（引入→冲突→转折→升华）
  │   ├── 清单体（N个理由/N个技巧）
  │   └── 场景体（痛点场景→产品介入→美好结果）
  ├── AI 优化开篇钩子：
  │   ├── 抖音：3 秒留人钩子
  │   ├── 小红书：首行痛点逻辑
  │   └── 公众号：悬念式/共鸣式开头
  └── 同步生成 3~5 版差异化标题
        ↓
[阶段 3] 润色层
  ├── 一键多平台语体改写
  │   ├── 小红书：口语化 + emoji + 体验感 + 话题标签
  │   ├── 抖音口播：短句 + 节奏标注 + 画面提示
  │   ├── 公众号：段落结构 + 引导关注 + 深度感
  │   └── Instagram：视觉描述 + hashtag 策略
  ├── 植入行业 SEO/GEO 关键词
  ├── 去 AI 味打磨（见 4.2.2）
  └── 自动违禁词合规校验
        ↓
[输出] 最终文案 + 操作卡片
```

#### 4.2.2 去 AI 味引擎

**三个档位，默认「自然」**：

| 档位 | 效果 | 适用场景 |
|------|------|---------|
| 自然（默认） | 去除 AI 套话，加入口语化、情绪细节、生活化表达 | 小红书、抖音、朋友圈 |
| 专业 | 保留结构清晰度，适度润色，减少过度口语化 | 公众号文章、官网文案 |
| 原味 | 仅做基本纠错，保留原始 AI 输出 | 用户想自己改或做 A/B 对比 |

**AI 味检测清单**（作为 Dify Prompt 约束注入）：

| AI 味特征 | 人类改写方向 |
|-----------|------------|
| "在当今...的时代/背景下" | 直接切入，删除时代背景铺垫 |
| "不仅...而且..." 并列套娃句 | 拆成短句，每句只讲一件事 |
| "值得注意的是/值得一提的是" | 直接说事实，删除引导语 |
| 每段都是"总-分-总"结构 | 打破规律结构，加入情绪转折 |
| 形容词堆砌（"卓越的""出色的""无与伦比的"） | 用具体数字和场景替代 |
| "为您带来/提供/打造" | 换成用户视角的具体效果 |
| 段末总是"让我们一起..." | 换成具体行动或留白 |
| 缺乏具体的人/时间/地点锚点 | 加入真实细节（"上周三...""北京的张姐..."） |
| 感叹号密度过高 | 控制在每 100 字 ≤ 2 个 |
| 排比句超过 3 组 | 最多保留 2 组，其余拆散 |

#### 4.2.3 兼容存量功能

- 批量草稿生成（保留）
- AI 局部定点修改（保留，增强为对话式：选中文字 → "把这段改成..."）
- 历史文案配置一键复用（保留）
- 生成后底部操作按钮：复制 | 润色 | 重新生成 | 存入草稿 | 定时发布

#### 4.2.4 环环相扣联动

文案生成完成后，AI 主动提议：
- "要不要基于这篇文案，帮你生成配套的小红书图文？"
- "要不要把核心卖点扩展成一版短视频脚本？"

### 4.3 图片生成模块【P2，第 6-7 周】

**核心职责**：降低图片生成门槛，从「写 prompt」变为「选风格 + 描述主体 + AI 自动补全」。

#### 4.3.1 生成流水线

```
用户意图 + Brief
        ↓
[步骤 1] 风格选择
  对话中展示风格卡片（带缩略图预览）：
  ┌──────────┬──────────┬──────────┬──────────┐
  │ 极简白底 │ 国潮插画 │ 3D 渲染  │ 赛博朋克 │
  │ [缩略图] │ [缩略图] │ [缩略图] │ [缩略图] │
  └──────────┴──────────┴──────────┴──────────┘
  ┌──────────┬──────────┬──────────┬──────────┐
  │ 日系清新 │ 轻奢质感 │ 科技感   │ 手绘风格 │
  │ [缩略图] │ [缩略图] │ [缩略图] │ [缩略图] │
  └──────────┴──────────┴──────────┴──────────┘
  （50+ 风格模板，分类展示）
        ↓
[步骤 2] 主体描述
  "画面里有什么？"（自然语言，无需专业术语）
  支持上传参考图（单图，AI 解析构图/光影/产品形态后复刻风格）
        ↓
[步骤 3] AI Prompt 增强
  后端将「自然语言 + 风格选择 + 品牌视觉规范」合成专业 Prompt
  展示优化后的 Prompt（默认折叠，高级用户可展开手动编辑）
  生成 3 种不同风格解读的提示词供预览选择
        ↓
[步骤 4] 多模型智能路由
  ├── 快速预览模式：轻量模型，<3s 出低清预览
  ├── 用户确认后：最佳质量模型生成高清图
  │   ├── 产品实拍图 → Flux/SD（可控性强）
  │   ├── 创意海报 → Midjourney（艺术性强）
  │   ├── 信息封面 → 内置模板引擎（品牌一致性）
  │   └── 快速迭代 → 轻量模型
  └── 过程展示进度动画："AI 正在构思画面... 25%"
        ↓
[步骤 5] 自动后处理
  ├── 品牌色统一调色
  ├── 封面卖点文字自动排版
  └── 一键输出全平台尺寸（1:1 / 3:4 / 9:16 / 16:9）
        ↓
[步骤 6] 对话式多轮调整
  图片下方快捷操作按钮：
  "背景更亮" | "去掉XX" | "换一个角度" | "生成变体"
  支持自然语言："把背景换成秋天傍晚的感觉"
        ↓
[步骤 7] 一键流转
  "保存至素材库" | "作为小红书配图" | "继续生成系列图" | "直接发布"
```

#### 4.3.2 素材短缺的三层应对

| 用户类型 | 素材情况 | 处理策略 |
|---------|---------|---------|
| 零素材 | 什么都没有 | AI 基于产品描述凭空生成场景草图 → 用户迭代 |
| 单素材 | 只有 1 张产品图 | AI 1 图衍生 20 种场景变体（换背景/角度/风格/光照） |
| 品牌用户 | 有完整素材库 | 自动调用品牌资产（Logo/产品图/品牌色）融入生成 |

#### 4.3.3 品牌资产自动抓取

输入品牌官网 URL → 自动提取：
- Logo（识别 header/favicon 中的品牌标识）
- 产品图（识别产品展示区域的图片）
- 品牌配色（提取 CSS 主色/辅色）
- 字体偏好（识别使用的字体族）

### 4.4 视频生成与智能剪辑模块【P2 L1 优先，P3 L2/L3】

**核心职责**：零素材也能成片，有素材 AI 辅助精剪。

#### 4.4.1 三阶能力分层

```
Level 1：模板化全自动短视频【P2，免费版可用】
  ├── 依据文案脚本自动生成
  ├── TTS 多音色配音（支持情感控制）
  ├── 从素材库/AI 生成匹配画面
  ├── 动态字幕（样式化、动画化）
  ├── 情绪匹配 BGM + 自动卡点转场
  ├── 零素材也能成片（全 AI 生成画面）
  └── 输出：单版本短视频

Level 2：AI 智能辅助剪辑【P3，Pro/企业版】
  ├── 用户上传实拍原片
  ├── AI 智能识别高光片段 + 剔除冗余镜头
  ├── 一键生成多版本：15s 竖版 / 60s 横版 / 9:16 / 16:9
  ├── 对话式修改："第三段换XX镜头""结尾加Logo"
  ├── 自动补齐空镜（素材不足时自动调取商用空镜）
  └── 版本管理：每次修改自动生成版本，可回溯对比

Level 3：全 AI 原创成片【P3 远期储备，企业定制】
  ├── AI 生成分镜脚本
  ├── 逐镜头 AI 生成画面（Sora/Runway/Kling 等前沿模型）
  ├── 品牌专属数字人出镜
  ├── 自动拼接剪辑
  └── 输出：全 AI 原创视频
```

#### 4.4.2 L1 模板化视频生成流程

```
用户输入 + Brief
        ↓
[步骤 1] AI 生成脚本与分镜卡片
  对话中展示可编辑的分镜卡片：
  ┌─────────────────────────────────┐
```

### 4.5 素材资产层（贯穿全模块，持续迭代）

#### 三级素材体系

```
L1 公共素材池
  ├── 平台采购全品类商用图片/视频/音频
  ├── AI 生成的背景/纹理/元素
  ├── 正版免费图库集成（Unsplash/Pexels 等，CC0 协议）
  └── 按会员分级开放（免费版限基础素材）

L2 用户素材智能扩容
  ├── 1 张原图 → AI 衍生 20 种场景变体
  ├── 少量素材 → AI 插值生成多版本
  └── 生成内容自动标注版权归属

L3 企业私有素材库
  ├── 企业版支持上传自有素材
  ├── AI 学习品牌素材风格 → 生成内容统一品牌视觉
  └── 品牌资产自动沉淀：生成的优质文案/图片/视频自动入库
```

#### 品牌资产自动沉淀

用户在系统中：
- 首次设置时上传的品牌素材（Logo/产品图/品牌色/文案规范）→ 入库
- 每次生成的内容，用户标记为"满意"的 → 自动入库作为正向样本
- 每次创作时的风格/语气偏好 → 持续学习，越用越贴合

### 4.6 质量管控层（全链路，全版本标配）

```
[生成前] 策略层数据质量
  ├── 竞品数据新鲜度检查（时效性 > 7 天 → 重新抓取）
  └── 品牌信息完整性检查（缺失字段 → 提示用户补充）

[生成中] 过程监控
  ├── Prompt 质量评分（是否包含所有必要要素）
  └── 生成进度追踪（各模块耗时监控）

[生成后] AI 质量自动打分
  ├── 文案评分维度：
  │   ├── 平台适配度（字数/格式/标签符合规范）
  │   ├── 品牌一致性（语气/禁用词/固定话术匹配）
  │   ├── 原创性（与训练数据/历史内容的重叠度）
  │   └── 营销有效性（CTA 清晰/钩子吸引力/结构完整）
  ├── 图片评分维度：
  │   ├── 品牌视觉一致性
  │   ├── 构图质量
  │   └── 主体清晰度
  └── 低分内容自动推荐 AI 二次优化

[用户端] 多版本横向对比
  ├── 同需求多版内容并排预览
  ├── 用户择优 → 选中的版本成为后续生成的风格参考
  └── 对比结果反馈给模型（强化学习信号）

[发布前] 平台规则校验
  ├── 54 平台发布规范（分层维护，Tier1 手动维护 + Tier2/3 自动爬取）
  ├── 限流关键词检测
  ├── 内容尺寸/时长/格式校验
  └── 违禁词合规检查

[人工微调] 任意环节支持
  ├── 用户手动修改 → AI 理解修改意图 → 二次润色
  ├── 非覆盖式编辑（保留原始版本）
  └── 修改历史可追溯
```
