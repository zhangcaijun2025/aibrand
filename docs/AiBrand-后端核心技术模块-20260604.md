## 一、Agent 感知层（核心差异化模块）

### 1.1 问候与上下文 API

| 方法 | 路由 | 功能 |
|------|------|------|
| GET | `/api/agent/greeting` | **Agent 问候** — 时间感知 + 姓名 + 系统状态 + 简报卡片 + 建议 + 隔夜事件 + 记忆上下文 |
| GET | `/api/agent/events` | **系统事件** — 未读的治愈/洞察/里程碑事件 |
| GET | `/api/agent/context` | **用户上下文** — 追踪的竞品/项目/偏好平台/风格/格式 |
| POST | `/api/agent/context` | 更新用户上下文 |
| GET | `/api/agent/profile` | **用户画像** — 行业/角色/里程碑/优势/回避主题 |
| POST | `/api/agent/behavior` | **行为追踪** — 前端埋点上报 |

**问候 API 返回数据结构**:
```typescript
{
  greeting: string          // "上午好"
  userName: string
  systemStatus: {           // 系统健康
    totalComponents: number
    healthyComponents: number
    uptime: string
    alerts: number
    asyncTasksRunning: boolean
  }
  memoryContext: {          // Agent 记忆
    projectCount: number
    competitorCount: number
    lastTopic: string
    isReturning: boolean
    daysSinceLastVisit: number
  }
  briefCards: []            // 简报卡片（配额/项目/竞品）
  suggestions: []           // AI 建议（可执行）
  overnightBrief: {         // 隔夜简报
    hasEvents: boolean
    summary: string
    events: []
  }
  isFirstVisit: boolean
  settlingIn: boolean       // 适应期标记
}
```

### 1.2 进化引擎 API

| 方法 | 路由 | 功能 |
|------|------|------|
| GET | `/api/agent/evolution/report` | **进化报告** — "Agent 学到了什么"（偏好/习惯/洞察/里程碑） |
| POST | `/api/agent/evolution/module-weights` | **模块权重** — 四因子排序（频率40%+新近度30%+上下文20%+完成率10%） |
| POST | `/api/agent/evolution/layout-check` | **布局检测** — Kendall Tau 距离 > 0.3 时推荐重排 |
| POST | `/api/agent/evolution/sync-profile` | **画像同步** — 从行为数据更新用户画像 |

### 1.5 MongoDB 数据模型

| Schema | 用途 |
|--------|------|
| SystemEvent | 系统事件（治愈/洞察/里程碑）+ 类型标记 |
| UserContext | 用户上下文（竞品/项目/偏好/会话历史） |
| UserProfile | 用户画像（行业/角色/里程碑/优势/回避主题/引导状态） |
| UserBehavior | 行为追踪（页面/动作/时间戳） |
| AgentDefinition | Agent 定义（名称/头像/性格/唤醒词/能力范围） |
| ComponentDefinition | 组件定义（类型/配置/市场信息） |
| UserInstalledComponent | 用户安装的组件状态 |
