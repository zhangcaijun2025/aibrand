## 七、核心技术数据流 (请求链路)

```
┌──────────────────────────────────────────────────────────────────────────┐
│                       一次完整的发布请求链路                                │
│                                                                          │
│  1. 用户点击「确认发送」                                                    │
│       │                                                                  │
│  2.   ▼  fetch('/api/publish/agent', { body: { platforms, content } })   │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │  Next.js App Router (BFF)                                        │    │
│  │  src/app/api/publish/agent/route.ts                              │    │
│  │                                                                  │    │
│  │  a. 验证 JWT Token (Authorization header)                        │    │
│  │  b. 校验 platforms[] + content (必填)                             │    │
│  │  c. 生成 taskId (agent_<timestamp>_<random>)                     │    │
│  │  d. pushPublishTask(taskId, url) → 任务队列                       │    │
│  │  e. 返回 { success: true, data: { taskId, status: 'queued' } }   │    │
│  └──────────────────────────────────────────────────────────────────┘    │
│       │                                                                  │
│  3.   ▼  Extension Polling (当前不可用)                                    │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │  Extension 定时 Ping: GET /api/extension/ping                     │    │
│  │  → 返回 { action: 'NEW_TASK', url }                               │    │
│  │  → Extension 打开发布 URL                                          │    │
│  │  → Extension 逐平台执行发布 + 上报进度                              │    │
│  └──────────────────────────────────────────────────────────────────┘    │
│       │                                                                  │
│  4.   ▼  远期: 异步队列方案                                                │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │  BullMQ Producer → Redis Queue → Worker Consumer                  │    │
│  │  → PlatformAdapter.publish(content, platform)                     │    │
│  │  → MongoDB 回写结果                                                │    │
│  └──────────────────────────────────────────────────────────────────┘    │
│       │                                                                  │
│  5.   ▼  前端轮询 / WebSocket 推送                                        │
│  ┌──────────────────────────────────────────────────────────────────┐    │
│  │  前端定时查询: GET /api/publish/tasks/:taskId                      │    │
│  │  → 返回发布进度 / 结果                                              │    │
│  │  → 更新前端 FEEDBACK 阶段 UI                                       │    │
│  └──────────────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────────────┘
```

## 十、技术栈总览

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Backend Stack                                   │
│                                                                     │
│  Runtime:      Node.js 20.18.3 (fnm)                                │
│  Framework:    NestJS (Nx Monorepo)                                 │
│  Language:     TypeScript (strict)                                  │
│  Database:     MongoDB 8.0 (Mongoose ODM)                           │
│  Cache:        Redis 7.x                                            │
│  Queue:        Redis List (BullMQ 远期)                              │
│  File Storage: RustFS (:9001)                                       │
│  AI Platform:  Dify 1.13.3 (:5001, 9 containers)                    │
│  Workflow:     n8n 2.18.4 (:5678, 7 激活工作流)                      │
│  LLM Bridge:   LangChain (:4010)                                    │
│  Gateway:      nginx (:8080, 限流+HSTS+CSP)                         │
│  Auth:         JWT + Redis 验证码                                    │
│  Validation:   Zod DTO Schema                                       │
│  Container:    Docker Compose (23 containers)                        │
│  Frontend:     Next.js 16 (BFF Pattern, :3099)                       │
│  Extension:    Plasmo MV3 (WXT v3 备用)                              │
└─────────────────────────────────────────────────────────────────────┘
```
