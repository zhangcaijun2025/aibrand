# AI选股系统部署指南

## 🚀 快速开始

### **第一步：恢复n8n服务**
1. **双击运行**：`快速启动n8n.bat`（桌面）
2. **选择方案**：如果工具弹出选择，选择方案1（Docker快速启动）
3. **等待启动**：等待约15秒服务启动
4. **验证访问**：打开浏览器访问 `http://localhost:5678/`
5. **登录凭证**：admin / Jun@2026

### **第二步：导入工作流**
1. **登录n8n**：使用上述凭证登录
2. **创建工作流**：点击左侧菜单"Workflows" → "New workflow"
3. **导入工作流**：
   - 点击右上角"Import from file"
   - 选择桌面上的工作流文件：
     - `AI选股系统_数据采集工作流.json`
     - `AI选股系统_AI分析工作流.json`
4. **激活工作流**：点击右上角"Activate"开关

### **第三步：配置数据库**
1. **安装PostgreSQL**（如果未安装）：
   ```bash
   # 使用Docker安装
   docker run -d --name postgres -p 5432:5432 -e POSTGRES_PASSWORD=Jun@2026 postgres
   ```

2. **创建数据库表**：
   ```sql
   -- 股票价格表
   CREATE TABLE stock_prices (
     id SERIAL PRIMARY KEY,
     code VARCHAR(20) NOT NULL,
     name VARCHAR(100),
     open DECIMAL(10,2),
     close DECIMAL(10,2),
     current DECIMAL(10,2),
     high DECIMAL(10,2),
     low DECIMAL(10,2),
     volume BIGINT,
     amount DECIMAL(20,2),
     timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   );
   
   -- 股票分析表
   CREATE TABLE stock_analysis (
     id SERIAL PRIMARY KEY,
     code VARCHAR(20) NOT NULL,
     name VARCHAR(100),
     total_score INTEGER,
     technical_score INTEGER,
     fundamental_score INTEGER,
     recommendation VARCHAR(50),
     action VARCHAR(10),
     current_price DECIMAL(10,2),
     analysis_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   );
   ```

3. **配置n8n数据库连接**：
   - 在n8n中：Settings → Credentials → Add Credential
   - 选择"PostgreSQL"
   - 配置信息：
     - Host: localhost
     - Port: 5432
     - Database: postgres
     - User: postgres
     - Password: Jun@2026

## 📊 工作流说明

### **工作流1：数据采集工作流**
```
定时触发器 → 股票数据API → 数据解析器 → 数据库写入 → 微信通知
```

**功能**：
- 每5分钟采集一次股票数据
- 支持5只股票：上证指数、深证成指、贵州茅台、五粮液、比亚迪
- 数据保存到PostgreSQL数据库
- 采集完成后发送微信通知

### **工作流2：AI分析工作流**
```
定时触发器 → 获取历史数据 → 技术分析 → 基本面分析 → 综合评分 → 保存结果 → 发送报告
```

**功能**：
- 每小时分析一次股票数据
- 技术面分析：移动平均线、RSI、MACD、布林带
- 基本面分析：PE、PB、ROE、股息率
- 综合评分：技术面60% + 基本面40%
- 生成投资建议：买入/卖出/观望
- 发送分析报告到微信

## 🔍 故障排除

### **常见问题1：n8n服务无法启动**
**症状**：`http://localhost:5678/` 无法访问

**解决方案**：
1. 检查端口占用：`netstat -ano | findstr :5678`
2. 停止占用进程：`taskkill /f /pid <PID>`
3. 重新启动n8n：`n8n start`
4. 或使用Docker：`docker run -d -p 5678:5678 n8nio/n8n`

### **常见问题2：数据库连接失败**
**症状**：工作流执行时数据库错误

**解决方案**：
1. 检查PostgreSQL服务是否运行
2. 验证数据库连接信息
3. 检查防火墙设置
4. 确认数据库表已创建

### **常见问题3：Python节点执行失败**
**症状**：Python脚本执行错误

**解决方案**：
1. 检查Python环境配置
2. 验证Python依赖是否安装
3. 检查脚本语法错误
4. 查看n8n Python节点日志

### **常见问题4：API调用失败**
**症状**：股票数据获取失败

**解决方案**：
1. 检查网络连接
2. 验证API地址是否正确
3. 检查API频率限制
4. 考虑使用备用数据源
