# AiBrand 用户自适应层 — 越用越懂你的 AI 伙伴

> 基于 Evolution Engine :4030，新增用户感知与 Skill 自动生成能力

---

## 一、核心理念

```
传统工具:  用户  → 学习工具 → 使用工具 → 适应工具
本系统:    系统  → 学习用户 → 适应用户 → 主动服务 → 创造工具
```

**目标**: 每多一次交互，系统就更懂用户一分。不是用户适应系统，是系统适应用户。

---

## 二、用户认知五维模型

系统持续构建用户的五维认知画像：

```
维度1: 需求图谱 (Need Map)
  高频需求 → 快捷入口
  复杂需求 → Agent 自动编排
  未满足需求 → 自动生成新工具

维度2: 行为节律 (Behavior Rhythm)
  活跃时段 → 定时预加载
  工作流顺序 → 智能预测下一步
  中断模式 → 主动提醒未完成任务

维度3: 风格偏好 (Style Profile)
  语言风格 (专业/轻松/极简)
  输出格式 (表格/文案/图表)
  决策模式 (快速/谨慎/数据驱动)

维度4: 技能缺口 (Skill Gap)
  用户想做什么但系统做不到 → 自动分析
  → 用现有组件能拼出来吗? → 生成工作流
  → 需要新能力吗? → 触发工具创造

维度5: 满意信号 (Satisfaction Signal)
  重复使用 → 隐性好评
  主动分享 → 强烈好评
  请求修改 → 需改进
  放弃任务 → 体验差
  直接反馈 → 显性信号
```

## 三、Skill 自动生成流水线

### 3.1 触发条件

```
同一需求出现 ≥ 3 次 → 生成快捷模板
用户说"能不能..." → 标记为技能需求
用户反复手动做某事 → 分析自动化可能
用户抱怨/放弃 → 触发体验优化
```

### 3.2 自动生成流程

```
Step 1: 需求捕获
  监听用户交互 (Dify 对话 / n8n 触发 / OpenClaw 消息)
  → 提取: { intent, entities, constraints, desired_output }

Step 2: 模式聚合
  相似需求聚合 → 聚类分析
  → 识别高频模式 → 排序优先级

Step 3: Skill 生成
  高优先级需求 → Claude Code 生成 Skill 定义
  → 包含: { name, trigger, tools, workflow, output_format }
  → 输出: SKILL.md + n8n workflow JSON

Step 4: 沙箱验证
  新 Skill 在隔离环境测试
  → 用历史数据回放 → 验证输出质量
  → 通过 → 部署到生产

Step 5: 用户感知
  下次用户提出类似需求 → 新 Skill 自动响应
  → 通知用户: "我学会了自动处理这类需求 ✨"
  → 收集反馈 → 继续优化
```

### 3.3 Skill 模板

```markdown
---
name: auto-generated-{timestamp}
description: 自动生成: 根据{user}的{need}创建
triggers:
  - 用户说 "{phrase_1}"
  - 用户说 "{phrase_2}"
tools:
  - {tool_from_analysis}
workflow: n8n/aibrand/{generated_workflow_id}
quality_gate:
  min_eval_score: 7
  auto_retry: true
---
```

## 四、用户感知触发点

### 4.1 主动感知

```
每日首次交互:
  → "早上好！今天为你准备了："
  → 昨日工作总结 / 今日待处理 / 新能力预告

感知到用户在工作流中:
  → 分析当前上下文 → 预测下一步
  → "需要我帮你把这份分析一键生成PPT吗？"

感知到用户重复劳动:
  → "我注意到你本周已手动做了 5 次竞品分析。
     我已学会自动处理这类需求，下次只需说'跑竞品分析'即可。"

感知到系统升级:
  → "我昨晚进化了：学会了 2 个新技能，优化了 3 个 Prompt。
     想看看吗？"
```

### 4.2 被动响应

```
用户说"要是能...就好了" → 捕获需求 → 评估可行性 → 自动生成 → 通知完成
用户说"这个不对" → 记录反馈 → 分析原因 → 自动修正 → 请求确认
用户说"太慢了" → 性能分析 → 自动优化 → 下次更快
用户说"帮我想想" → 创意模式 → 生成多个方案 → 让用户选择
```
